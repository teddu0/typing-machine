# Клавишки

Дружелюбный тренажёр слепой десятипальцевой печати для детей.

## Запуск

Для полного запуска с регистрацией и серверным прогрессом рекомендуется Docker
Compose:

```bash
docker compose up --build -d
```

Команда поднимает приложение и PostgreSQL. При старте приложение автоматически
применяет SQL-миграции. Затем откройте
[http://localhost:3000](http://localhost:3000).

PostgreSQL доступен с компьютера по адресу из `DATABASE_URL`. Параметры
подключения и внешний порт можно изменить в `.env`.

Для разработки без Docker нужен доступный PostgreSQL и переменная
`DATABASE_URL`. Пример структуры переменных находится в `.env.example`; реальные
значения храните только в локальном `.env`.

```bash
npm install
npm run migrate
npm start
```

Затем откройте [http://localhost:3000](http://localhost:3000).

Интерактивная документация backend API доступна в Swagger UI:
[http://localhost:3000/api/swagger](http://localhost:3000/api/swagger).
OpenAPI-схема доступна по адресу
[http://localhost:3000/api/openapi.json](http://localhost:3000/api/openapi.json).

## Запуск через Docker на другом компьютере

На компьютере должны быть установлены Git, Docker и Docker Compose.

Склонируйте проект и перейдите в его папку:

```bash
git clone <ссылка-на-репозиторий>
cd typing-machine
```

Соберите и запустите приложение в фоне:

```bash
docker compose up --build -d
```

После запуска откройте [http://localhost:3000](http://localhost:3000).

Для доступа с другого устройства в той же локальной сети откройте:

```text
http://IP-АДРЕС-КОМПЬЮТЕРА:3000
```

Полезные команды:

```bash
# Посмотреть состояние контейнера
docker compose ps

# Посмотреть логи
docker compose logs -f

# Остановить приложение
docker compose down

# Обновить приложение после изменений
git pull
docker compose up --build -d

# Полностью пересоздать БД и применить миграции заново
# Внимание: команда безвозвратно удаляет пользователей и прогресс
docker compose down -v
docker compose up --build -d
```

Учебный маршрут включает средний, верхний и нижний буквенные ряды, цифры и
большой курс практики с постепенным смешиванием символов.

Упражнения можно выполнять с физической или экранной клавиатуры. Прогресс
сохраняется в браузере автоматически.

Пользователь может зарегистрироваться или войти по email и паролю. После
авторизации локальный гостевой прогресс объединяется с серверным прогрессом в
PostgreSQL и далее синхронизируется автоматически.

В приложении есть страница рейтинга. В неё попадают авторизованные участники,
у которых есть серверный прогресс или история завершённых занятий. В списке
отображается имя из профиля, а если оно не заполнено — email. Телефон и дата
рождения не показываются.
Очки рейтинга начисляются за завершённые занятия, звёзды и полные минуты
практики.

Авторизованный пользователь может открыть профиль, указать необязательные имя,
дату рождения и телефон, а также изменить пароль. После смены пароля остальные активные
сессии аккаунта завершаются.

В профиле можно безвозвратно обнулить серверный и локальный прогресс. Операция
требует дополнительного подтверждения словом `обнулить`.

Экранные клавиатуры используют мягкие цветовые зоны правильных пальцев.
Отдельная инструкция показывает постановку рук и связь пальцев с клавишами.
После занятия кнопки результата можно выбирать стрелками и нажимать пробелом.

## Проверка проекта

```bash
npm run check
```

Команда проверяет синтаксис, учебные данные, сохранение прогресса и основную
клиентскую и серверную логику.

## Backend API

Backend написан на Node.js без серверного фреймворка. HTTP-маршрутизация,
авторизация, профиль и прогресс разделены на модули в `server/`. Для PostgreSQL
используется драйвер `pg`, а SQL-миграции из `migrations/` автоматически
применяются перед запуском сервера.

Основные группы API:

- `/api/auth/*` — регистрация, вход, выход и текущая сессия.
- `/api/profile*` — данные профиля и смена пароля.
- `/api/progress*` — получение, объединение и обнуление прогресса.
- `/api/leaderboard` и `/api/typing-sessions` — рейтинг и запись завершённых занятий.
- `/api/courses` — учебные курсы.
- `/api/health` — проверка приложения и PostgreSQL.
- `/api/swagger` и `/api/openapi.json` — интерактивная документация и схема API.

## Переменные окружения

- `DATABASE_URL` — строка подключения к PostgreSQL.
- `POSTGRES_DB` — имя базы данных PostgreSQL.
- `POSTGRES_USER` — пользователь PostgreSQL.
- `POSTGRES_PASSWORD` — пароль пользователя PostgreSQL.
- `POSTGRES_PORT` — внешний порт PostgreSQL в Docker Compose, по умолчанию `5432`.
- `NODE_ENV` — `development` или `production`.
- `PORT` — порт приложения, по умолчанию `3000`.
- `SESSION_COOKIE_NAME` — имя cookie серверной сессии.
- `SESSION_DAYS` — срок действия сессии в днях.

В production необходимо заменить пароли PostgreSQL в `.env`, включить HTTPS и
хранить секреты вне репозитория.

## Полный доступ для тестирования

Чтобы временно открыть все курсы и занятия, измените флаг в `public/app.js`:

```js
const isFullAccess = true;
```

При значении `false` занятия открываются последовательно по мере прохождения.
