# Контекст проекта

## Идея

«Клавишки» — браузерный тренажёр слепой десятипальцевой печати для детей,
которые только начинают знакомиться с компьютером.

## Главные принципы

- Интерфейс должен быть добрым, спокойным и понятным без инструкции.
- Ошибка — это подсказка, а не наказание.
- Новые буквы добавляются постепенно и часто повторяются.
- Крупные элементы, короткие тексты и минимум отвлекающих действий.
- Прогресс сохраняется автоматически.
- Ошибочная английская раскладка не считается ошибкой ученика: интерфейс
  показывает `EN` и помогает переключиться на `RU`.

## Текущий учебный маршрут

Маршрут состоит из пяти последовательных курсов:

1. Средний ряд — 36 занятий.
2. Верхний ряд — 22 занятия.
3. Нижний ряд — 18 занятий.
4. Цифры — 15 занятий.
5. Практика — 24 занятия с постепенным смешиванием рядов, слов и цифр.

Опорные клавиши: `а` и `о`. На них обычно находятся метки для указательных
пальцев. В экранной клавиатуре эти насечки обязательно отображаются.

Курс строится постепенно: отдельная клавиша, соседняя пара, небольшое
смешивание и закрепление. Новые буквы не должны появляться раньше, чем ребёнок
несколько раз повторил уже изученные движения. Текущая версия курса содержит
короткие уровни. Следующий курс открывается после завершения предыдущего.

## Техническая архитектура

- Backend: модульный Node.js API и PostgreSQL.
- Внешняя серверная зависимость: официальный PostgreSQL-драйвер `pg`.
- Модули Node.js: ES Modules.
- API разделён на группы `auth`, `profile`, `progress`, `courses` и `health`;
  актуальная интерактивная документация доступна в `/api/swagger`.
- Frontend: HTML, CSS и JavaScript.
- Клиентский JavaScript разделён на модули: `app.js` управляет экранами и
  занятием, `keyboard-layouts.js` описывает раскладки и пальцы, `progress.js`
  отвечает за сохранение и миграции прогресса.
- Активная раскладка подключается в `app.js` как конфигурация. Это позволяет
  добавить английский маршрут без дублирования логики клавиатуры и тренажёра.
- Favicon повторяет логотип приложения: фиолетовая клавиша с буквой `А`.
- Браузер не предоставляет системную раскладку напрямую. Интерфейс определяет
  `RU` или `EN` по последней нажатой буквенной клавише.
- Гостевой прогресс хранится в `localStorage`; после регистрации он
  объединяется с серверным прогрессом в PostgreSQL.
- Версия учебного курса хранится вместе с прогрессом, чтобы изменения порядка
  уровней не связывали старые звёзды с новыми упражнениями.
- Прогресс хранится по ключу `courseId:levelId`. При переходе на модель курсов
  звёзды старого среднего ряда сохраняются.
- Клиентский флаг `isFullAccess` в `public/app.js` открывает все курсы и занятия
  для ручного тестирования. В обычном режиме он должен быть `false`.
- Буква `Ё` не входит в текущий учебный маршрут и не показывается на экранной
  клавиатуре.
- В хедере доступна отдельная инструкция по постановке рук. Она показывает
  домашний ряд, насечки `А` и `О`, положение больших пальцев над пробелом и
  напоминает держать кисти расслабленными.
- Инструкция содержит интерактивную полную учебную клавиатуру: при наведении
  или фокусе нужный палец подсвечивается, а подпись называет палец и руку.
- Статичные руки в инструкции расположены под клавиатурой и не закрывают
  клавиши. Пальцы окрашены в те же цвета, что и соответствующие группы
  клавиш.
- Экранные клавиатуры используют лёгкую симметричную цветовую кодировку:
  одинаковые пальцы левой и правой руки имеют одинаковый оттенок клавиш.
- Инструкция объединяет клавиатуру, руки и три короткие подсказки в одной
  учебной зоне. На широком экране подсказки находятся рядом со схемой, на
  узком — переходят под неё.
- Экран занятия имеет компактные режимы для невысоких ноутбучных экранов:
  клавиатура и текст упражнения помещаются в область просмотра без
  предварительной прокрутки.
- На экране результата кнопки можно выбирать стрелками влево и вправо и
  нажимать пробелом. По умолчанию выбрано следующее занятие.
- На экране занятия отображается таймер. После завершения урока время
  сохраняется в истории занятия и учитывается в рейтинге.
- Глобальный ввод занятия приостанавливается при открытом модальном окне
  авторизации и не перехватывает ввод из полей формы.
- Текущий информационный экран сохраняется в hash URL: карта, инструкция,
  рейтинг и профиль восстанавливаются после обновления страницы. Активное
  занятие после refresh намеренно возвращается к карте, потому что состояние
  текущего упражнения хранится только в памяти.
- Контейнеризация: Docker Compose.
- Docker Compose поднимает приложение и PostgreSQL; SQL-миграции применяются
  автоматически при запуске приложения.
- Для запуска обязателен локальный `.env`; `.env.example` содержит только
  плейсхолдеры и не должен включать реальные креды. `DATABASE_URL` больше не
  имеет рабочего fallback-значения в коде.
- В Docker Compose PostgreSQL публикуется только на `127.0.0.1`, чтобы база не
  была доступна извне VPS. Наружу для тестового IP-доступа публикуется порт
  приложения `3000`.
- Для тестового доступа по `http://IP:3000` можно временно использовать
  `NODE_ENV=development`. Для публичного production-запуска нужен HTTPS, после
  чего можно включать `NODE_ENV=production`.
- Сервер отдаёт базовые security headers и CSP для статического фронтенда.
- Реализованы регистрация и вход по email и паролю без подтверждения почты.
  Пароли хешируются через `scrypt`, сессии хранятся на сервере, а браузер
  получает защищённую `HttpOnly` cookie.
- Авторизованный пользователь может указать необязательные отображаемое имя,
  дату рождения и телефон, а также изменить пароль с завершением остальных активных
  сессий.
- OpenAPI-схема и локальный Swagger UI документируют backend API.
- После регистрации или входа локальный гостевой прогресс объединяется с
  серверным прогрессом в PostgreSQL по максимальному числу звёзд.
- Завершения занятий без входа сохраняются в `localStorage` как
  `klavishki-pending-sessions`; после регистрации или входа они отправляются в
  `typing_sessions`, чтобы время гостевой практики попало в рейтинг.
- При выходе из аккаунта локальный прогресс и локальная история ожидающих
  занятий очищаются. Это защищает семейный сценарий, когда несколько
  пользователей работают за одним компьютером и браузером.
- Авторизованный пользователь может обнулить серверный и локальный прогресс
  после текстового подтверждения действия.
- Реализован MVP рейтинга участников. В таблицу попадают пользователи с
  серверным прогрессом или историей завершённых занятий; на UI показывается
  `display_name`, а если имя не заполнено — email. Телефон и дата рождения в
  рейтинге не отображаются.
- Очки рейтинга считаются по формуле: `10` очков за каждое уникально
  завершённое занятие, `3` очка за каждую звезду и `1` очко за каждую полную
  минуту практики.
- Завершения занятий авторизованных пользователей пишутся в отдельную таблицу
  `typing_sessions`. Это позволит позже добавить недельные рейтинги, серии,
  подробную динамику и статистику по буквам без изменения таблицы прогресса.
- Список участников рейтинга строится не только по `typing_sessions`, но и по
  таблице `progress`, потому что гостевой прогресс может переноситься в аккаунт
  без исторических записей отдельных занятий.

## Следующие важные решения

- До английского маршрута добавить месячную подписку и серверную модель прав
  доступа; аккаунты и серверный прогресс уже реализованы.
- Выбрать PostgreSQL-инфраструктуру, почтовый сервис и платёжного провайдера.
- Определить страну регистрации бизнеса и юридические требования к платежам,
  регулярным списаниям и обработке данных пользователей.
- Добавить экран для взрослого с подробной статистикой.
- Решить, нужны ли звук и озвучивание инструкций.
- Решить, добавлять ли знаки препинания и Shift отдельным курсом.
- Добавить отдельный учебный маршрут для английской раскладки и слепой печати
  на английском языке.
- Протестировать понятность инструкции с детьми и взрослыми на разных
  размерах экрана.

## План аккаунтов и подписки

- Гость получает инструкцию и курс среднего ряда. Его прогресс хранится
  локально.
- После завершения среднего ряда приложение предлагает месячную подписку.
- Для оплаты создаётся аккаунт взрослого по email и паролю.
- Активная подписка открывает остальные курсы и будущие платные возможности.
- Сервер становится источником истины для аккаунтов, прогресса, подписок и
  прав доступа.
- Подробная архитектура и порядок реализации описаны в
  [`ACCOUNTS_AND_SUBSCRIPTIONS.md`](./ACCOUNTS_AND_SUBSCRIPTIONS.md).

## Состояние на 13 июня 2026 года

- Работают пять русскоязычных курсов: средний ряд, верхний ряд, нижний ряд,
  цифры и практика.
- В маршруте 115 занятий; буква `Ё` намеренно исключена.
- Доступны последовательное открытие занятий и тестовый режим `isFullAccess`.
- Гостевой прогресс сохраняется локально; после регистрации он объединяется с
  серверным прогрессом в PostgreSQL.
- Экранные клавиатуры интерактивны, показывают насечки `А` и `О`, пробел и
  цветовые зоны правильных пальцев.
- Есть отдельная расширенная инструкция по постановке рук с цветной
  клавиатурой, статичными цветными пальцами и тремя короткими подсказками.
- Экран занятия адаптирован для невысоких ноутбучных экранов.
- Экран результата поддерживает выбор действий стрелками и пробелом.
- Клиентская логика раскладки и прогресса вынесена в отдельные ES-модули.
- Неверная английская раскладка определяется по вводу и сопровождается
  подсказкой переключиться на `RU`.
- Приложение запускается напрямую через Node.js или через Docker Compose.
- Работают регистрация, вход, серверные сессии и личный кабинет с именем,
  датой рождения в формате `ДД.ММ.ГГГГ`, телефоном и сменой пароля.
- Работают безопасное обнуление прогресса, автоматические SQL-миграции,
  OpenAPI-схема и Swagger UI.
- Добавлен первый рейтинг печатальщиков: он считает завершённые занятия,
  звёзды, среднюю точность, время практики и очки, а в списке отображает только
  имя участника или email, если имя не заполнено. Минуты практики теперь входят
  в формулу очков, а суммарное время показывается под именем в формате
  `минуты:секунды`.

## Деплой на VPS без домена

1. Склонировать репозиторий на VPS.
2. Создать `.env` из `.env.example` и заполнить реальные значения:
   `DATABASE_URL`, `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD`,
   `POSTGRES_PORT`, `NODE_ENV`, `PORT`, `SESSION_COOKIE_NAME`, `SESSION_DAYS`.
3. Для проверки по IP использовать `NODE_ENV=development`, чтобы cookie входа
   работала без HTTPS.
4. Запустить `docker compose up --build -d`.
5. Проверить `docker compose ps`, `docker compose logs -f app` и
   `curl http://localhost:3000/api/health` на сервере.
6. Если локально на VPS работает, а снаружи нет, проверить firewall провайдера
   и открыть входящий TCP-порт `3000`.
7. PostgreSQL наружу не открывать; в Compose он привязан к `127.0.0.1`.
